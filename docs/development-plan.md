# 宝宝成长记 MVP 开发计划

| 项目 | 内容 |
| --- | --- |
| 文档版本 | v0.1 |
| 计划方式 | 按里程碑推进，不绑定具体人数与日期 |
| 更新日期 | 2026-07-16 |

## 1. 计划目标

以微信小程序为首发端，交付可供家庭内部试用的 MVP：用户能够登录、创建宝宝、记录成长内容、查看时间轴和成长趋势、邀请家庭成员，并导出数据。

本文按依赖关系定义里程碑。具体排期应在团队规模和核心交互稿确定后补充。云服务商不阻塞本地开发，最晚在预发布环境建设前确定。

## 2. 范围基线

MVP 包含：

- 微信登录。
- 多宝宝档案。
- 图文、身高体重和里程碑记录。
- 成长时间轴与类型筛选。
- 身高、体重趋势。
- 家庭邀请和三级权限。
- 图片上传与私有访问。
- 数据导出基础流程。
- 隐私、安全、日志、备份和上线准备。

不包含喂养、睡眠、尿布、疫苗提醒、视频、AI 周报、社区和医疗诊断。

## 3. 开发原则

- 先完成纵向可运行链路，再逐步增加记录类型和管理能力。
- 每个里程碑同时包含客户端、服务端、数据迁移和测试，不把测试集中到最后。
- 所有写接口从第一天实施权限校验、参数校验和幂等策略。
- 微信专有能力必须经过平台适配层。
- 数据库迁移、OpenAPI 和关键文档与代码同步评审。
- 未完成的外部依赖使用明确的 mock，不在业务代码中散落临时假数据。

## 4. 里程碑概览

| 阶段 | 交付结果 | 依赖 |
| --- | --- | --- |
| M0 方案收敛 | 决策、原型、接口骨架可评审 | 当前文档 |
| M1 工程基础 | 客户端和 API 可本地运行、CI 可验证 | M0 |
| M2 登录与宝宝档案 | 用户可登录并创建/切换宝宝 | M1 |
| M3 成长记录与时间轴 | 可完成三类记录的完整生命周期 | M2 |
| M4 成长数据 | 身高体重趋势与历史列表可用 | M3 |
| M5 家庭协作 | 邀请、授权、移除和权限隔离可用 | M2、M3 |
| M6 导出与隐私 | 管理员可发起并下载导出包 | M3、M5 |
| M7 稳定性与发布 | 微信体验版通过验收，可提交审核 | M2-M6 |

## 5. M0：方案收敛

### 目标

解决会影响工程结构和返工成本的关键问题。

### 任务

- 确认产品正式名称、目标用户年龄段和首发范围。
- 确认使用 PostgreSQL 和 S3 兼容对象存储的技术边界；具体云服务商和部署区域可延后到预发布环境建设前确定。
- 确认微信小程序主体、AppID、服务器域名和隐私类目准备情况。
- 输出核心页面低保真原型和交互状态。
- 完成 OpenAPI 初稿，至少覆盖登录、宝宝、记录和上传。
- 确认图片大小、数量、压缩和原图策略。
- 确认数据导出格式和数据保留期限。
- 建立架构决策记录目录 `docs/adr/`，重大决策逐项记录。

### 完成标准

- 不存在阻塞工程初始化的待确认技术选项；云厂商选择不属于工程初始化阻塞项。
- 首页、添加记录、时间轴、成长数据和家庭成员流程可评审。
- 数据模型和首批 API 命名一致。

## 6. M1：工程基础

### 客户端

- 初始化 Taro + React + TypeScript 项目。
- 建立页面路由、底部导航和基础主题变量。
- 建立平台适配层、API 请求层和全局错误处理。
- 配置 Zustand、表单校验和测试环境。
- 实现加载、空状态、错误状态和确认弹窗等基础组件。

### 服务端

- 初始化 NestJS、Prisma 和 PostgreSQL。
- 建立配置管理、环境校验、结构化日志和请求 ID。
- 建立统一校验、异常响应、认证守卫和权限守卫骨架。
- 创建初始数据库迁移。
- 生成并发布 OpenAPI 文档。

### 工程质量

- 配置 pnpm workspace、TypeScript、ESLint 和格式化规则。
- 配置单元测试、API 测试和 CI。
- 提供 `.env.example`，真实密钥不得提交。
- 编写本地开发启动说明。
- 使用 Docker Compose 提供 PostgreSQL 和 MinIO；模拟登录在 M2 实现，M1 只保留安全的配置扩展点。

### 完成标准

- 新环境按 README 可以启动客户端和 API。
- CI 能完成静态检查、类型检查、单元测试和构建。
- 客户端可调用 API 健康检查并显示标准错误状态。

## 7. M2：登录与宝宝档案

### 功能任务

- 实现微信临时凭证换取和业务会话。
- 实现访问令牌刷新、退出和会话失效处理。
- 创建、查看和编辑宝宝档案。
- 创建宝宝时原子创建管理员成员关系。
- 获取用户可访问宝宝列表。
- 实现当前宝宝选择、恢复和切换。
- 实现无宝宝首次引导。
- 实现宝宝头像上传或先使用安全默认头像；具体依赖 M0 决策。

### 测试重点

- 相同微信身份重复登录不会创建重复用户。
- 未登录用户不能访问业务接口。
- 用户不能访问未加入的宝宝。
- 多宝宝切换后数据上下文正确。
- 宝宝出生日期和身高体重校验正确。

### 完成标准

- 新用户可从打开小程序走到宝宝首页空状态。
- 老用户重新登录后能恢复有效宝宝上下文。
- 权限越权测试通过。

## 8. M3：成长记录、图片与时间轴

### 服务端任务

- 实现媒体上传凭证、上传完成确认和临时访问地址。
- 实现图文、测量和里程碑记录的创建、读取、编辑和软删除。
- 实现记录与媒体关联的事务处理。
- 实现时间轴游标分页和类型筛选。
- 实现幂等键，避免弱网重复创建。
- 建立未关联媒体清理任务。

### 客户端任务

- 实现记录类型选择和三类表单。
- 实现选图、压缩、上传进度、失败重试和排序。
- 实现时间轴列表、筛选、分页和下拉刷新。
- 实现记录详情、编辑和删除确认。
- 按角色和创建者显示操作权限。

### 测试重点

- 文字为空但有图片的图文记录可以保存，二者皆空不可保存。
- 测量至少包含身高或体重。
- 里程碑标题必填。
- 图片上传失败不丢失表单，重试不重复创建记录。
- 分页在新增记录后保持稳定。
- 编辑成员不能修改他人记录，只读成员不能写入。

### 完成标准

- 三类记录均完成创建、查看、编辑和删除闭环。
- 时间轴正确展示混合记录和图片。
- 微信真机完成拍照/选图和上传验证。

## 9. M4：成长数据

### 任务

- 实现身高和体重时间范围查询。
- 实现趋势图、指标切换和数据点详情。
- 实现测量历史列表及编辑入口。
- 统一数值精度、单位和显示格式。
- 增加“非医疗诊断”说明。
- 验证图表在小程序端的包体、渲染和触摸体验。

### 测试重点

- 仅身高、仅体重和两者都有的记录均正确展示。
- 同一天多次测量按发生时间排列。
- 编辑或删除测量后图表和列表同步更新。
- 无数据、单点、大量数据和跨年数据状态可用。

### 完成标准

- 图表与历史记录数据一致。
- 真机性能达到可接受水平，无明显卡顿或溢出。

## 10. M5：家庭协作

### 任务

- 实现管理员生成、分享、撤销邀请。
- 实现邀请预览、登录后继续和确认加入。
- 实现邀请过期、单次使用和幂等接受。
- 实现成员列表、角色修改和成员移除。
- 实现最后一个管理员保护规则。
- 对宝宝、记录、媒体和导出接口执行统一成员权限校验。
- 为权限和成员变化写入审计日志。

### 测试重点

- 已过期、撤销和使用过的邀请不能重复加入。
- 同一用户并发接受邀请只生成一条成员关系。
- 被移除用户的既有令牌无法继续读取宝宝数据。
- editor、viewer 和 admin 权限矩阵全部覆盖。
- 最后一个管理员不能退出或被降级。

### 完成标准

- 两个真实微信账号可以完成邀请和协作流程。
- 客户端隐藏操作与服务端强制权限保持一致。
- 自动化越权测试通过。

## 11. M6：数据导出与隐私能力

### 任务

- 确定导出结构并实现导出任务表。
- 实现任务创建、处理、状态查询和失败重试。
- 生成档案、记录、测量和媒体清单；照片打包按 M0 决策执行。
- 将导出包写入私有对象存储并生成限时下载地址。
- 实现下载和敏感操作审计。
- 增加隐私政策入口、数据处理说明和注销/删除入口设计。

### 测试重点

- 非管理员不能导出。
- 导出只包含目标宝宝和可约定范围的数据。
- 下载地址过期后不可访问。
- 导出失败有可理解状态，重复创建受限流和幂等保护。

### 完成标准

- 管理员可以在测试环境完成一次完整导出并核对内容。
- 导出包及链接符合已确定的保留和过期规则。

## 12. M7：稳定性、安全与发布

### 稳定性

- 完成核心流程端到端测试。
- 验证弱网、超时、重复点击、上传中断和会话过期。
- 完成数据库备份与恢复演练。
- 完成对象存储生命周期规则验证。
- 建立生产监控、错误告警和关键指标看板。

### 安全与合规

- 对所有宝宝域接口进行越权检查。
- 完成依赖漏洞和密钥泄漏检查。
- 审查日志，确保不记录宝宝信息、正文、令牌和签名地址。
- 完成微信隐私保护指引、用户协议和隐私政策配置。
- 确认服务器域名、上传/下载域名及业务类目。

### 发布

- 创建测试、预发布和生产环境。
- 执行数据库迁移和回滚演练。
- 微信体验版由目标家庭用户验收。
- 修复阻塞和高优先级问题。
- 准备版本说明、审核材料和发布回滚方案。

### 完成标准

- PRD 的 MVP 验收标准全部通过。
- 无已知阻塞级和高风险安全问题。
- 生产备份、监控、告警和回滚均可操作。
- 微信小程序具备提交审核条件。

## 13. 横向工作流

以下工作不应等到单一里程碑集中处理：

### 文档

- 数据模型变更同步更新 `data-model.md` 和迁移。
- 路由或流程变更同步更新 `information-architecture.md`。
- 接口变更同步更新 OpenAPI。
- 重大技术决策添加 ADR。

### 质量

- 每个功能至少包含正常路径、校验失败和越权路径测试。
- 合并前必须通过 lint、类型检查、测试和目标端构建。
- 关键逻辑优先测试权限、事务和幂等，不只测试 UI 快照。

### 隐私

- 新增字段时说明收集目的、使用范围和保留期限。
- 新增第三方 SDK 前完成隐私和包体评估。
- 测试环境不使用真实宝宝照片和个人资料。

## 14. 工作项优先级

| 优先级 | 定义 |
| --- | --- |
| P0 | 阻塞核心流程、数据安全或发布，必须立即处理 |
| P1 | MVP 必需能力或严重体验问题，当前里程碑解决 |
| P2 | 有明确价值但不阻塞 MVP，可排入后续迭代 |
| P3 | 探索或优化项，进入候选池 |

需求新增必须先判断是否属于 MVP。若不属于，应更新候选功能而非插入当前里程碑。

## 15. Definition of Done

单个功能完成需要同时满足：

- 验收条件明确并通过。
- 权限、参数和异常路径已实现。
- 必要的数据库迁移可前进执行，并有回滚说明。
- OpenAPI、数据模型和用户流程文档已同步。
- 单元/集成测试覆盖核心规则。
- 微信小程序端完成至少一次真实运行验证。
- 日志和埋点不包含敏感内容。
- 无新增高等级静态检查或安全问题。

## 16. 风险与缓解

| 风险 | 影响 | 缓解措施 |
| --- | --- | --- |
| 微信审核和隐私要求准备不足 | 延迟发布 | M0 提前确认主体、类目和隐私接口 |
| 图片存储与流量成本不可控 | 成本上升 | 限制大小和数量、压缩、生命周期规则、成本告警 |
| 多端 API 差异 | 后期移植返工 | 平台适配层、禁止业务代码直接调用 `wx.*` |
| 家庭权限越权 | 严重隐私事故 | 服务端统一守卫、资源反查、越权自动化测试 |
| 弱网导致重复记录 | 数据质量下降 | 客户端锁、幂等键、事务和可恢复上传 |
| 图表组件包体或性能问题 | 无法上线/体验差 | M4 前做技术验证，准备轻量图表降级方案 |
| 导出任务耗时和内存过高 | 服务不稳定 | 异步任务、流式生成、大小限制、任务隔离 |

## 17. 下一步行动

1. 开发 agent 阅读开发交接说明、页面交互规格和 API 契约。
2. 初始化 pnpm workspace、Taro 客户端和 NestJS API。
3. 使用 Docker Compose 启动 PostgreSQL 与 MinIO，打通客户端调用 `/api/v1/health`。
4. M1 验收后，在 M2 实现“模拟登录 → 创建宝宝 → 首页空状态”业务链路。
5. 实现过程中同步生成 OpenAPI 3 文件，并将差异反馈到文档。
