# ADR-0003：宝宝资源实施服务端资源级授权

> [文档索引](../../README.md) · 分类：系统架构

- 状态：Accepted
- 日期：2026-07-16

## 背景

宝宝档案、照片和成长记录属于敏感家庭数据。一个用户可以访问多个宝宝，一个宝宝也可以有多个不同角色的成员。仅依赖客户端隐藏按钮或信任请求中的角色会产生严重越权风险。

## 决策

- 所有宝宝域请求先验证业务会话，再查询有效 `baby_members`。
- 详情、媒体和导出等只提供资源 ID 的接口，必须由服务端从资源反查 `baby_id` 后授权。
- 客户端传入的角色、创建者和 babyId 不能作为授权事实。
- admin、editor、viewer 权限在服务端统一策略中实现。
- 角色变更和成员移除立即影响后续请求，不依赖访问令牌中的缓存角色。
- 跨宝宝关联 mediaId、recordId、memberId 必须有自动化越权测试。

## 影响

正面影响：权限逻辑集中且可测试，成员被移除后无需等待令牌过期，降低横向越权风险。

代价：每个宝宝域请求需要一次成员关系校验；可通过合理索引和请求内缓存降低开销，不能通过跳过校验优化。

## 替代方案

- 把角色写入长期 JWT：角色变化无法立即生效，未采用。
- 仅在客户端控制权限：不能形成安全边界，未采用。
