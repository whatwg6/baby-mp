# ADR-0002：成长记录采用统一主表与类型详情表

> [文档索引](../../README.md) · 分类：系统架构

- 状态：Accepted
- 日期：2026-07-16

## 背景

MVP 包含图文、测量和里程碑三类记录，后续可能增加睡眠、喂养等类型。所有类型都需要进入同一时间轴，并共享权限、创建者、发生时间、媒体和软删除能力。

## 决策

- 使用 `records` 保存共有字段。
- 测量等强类型数据使用一对一详情表，例如 `measurement_records`。
- 媒体通过 `record_media` 与统一记录关联。
- `metadata` 只保存非核心扩展信息，不替代强类型字段。
- 新增记录类型时优先增加详情表，不向 `records` 堆积大量可空专有列。

## 影响

正面影响：时间轴查询统一，权限和审计只实现一次，新增类型的迁移边界清楚。

代价：读取部分类型需要连接详情表；创建记录和详情必须使用事务保证一致。

## 替代方案

- 每种记录独立表：时间轴需要联合多个表，分页和排序复杂，未采用。
- 所有专有字段存入 JSON：约束、查询和迁移较弱，未采用。
