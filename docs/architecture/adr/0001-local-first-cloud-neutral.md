# ADR-0001：本地优先且保持云厂商中立

> [文档索引](../../README.md) · 分类：系统架构

- 状态：Accepted
- 日期：2026-07-16

## 背景

产品以微信小程序首发，但具体云服务商、部署区域和 CDN 尚未确定。如果工程初始化依赖某家云平台，会延迟开发并增加后续迁移成本。

## 决策

- 本地开发使用 Docker PostgreSQL、MinIO 和模拟登录。
- 数据库面向标准 PostgreSQL 开发。
- 文件存储通过 S3 兼容适配器访问，业务层不调用厂商专属 SDK。
- 微信登录和分享通过平台适配层实现。
- 云服务商最晚在预发布环境建设前确定，不阻塞 M1 至 M4。
- staging 和 production 禁止启用模拟登录。

## 影响

正面影响：开发可立即开始，自动化测试可离线运行，云厂商替换成本较低。

代价：正式部署前必须验证目标对象存储与 MinIO 在签名、CORS、生命周期和回调行为上的差异；厂商专属能力需要通过独立适配器接入。

## 替代方案

- 直接使用微信云开发：初期快，但跨端和迁移约束更强，未采用。
- 在 M0 立即确定云厂商：对本地工程没有必要，未采用。
