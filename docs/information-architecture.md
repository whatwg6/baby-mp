# 宝宝成长记信息架构与核心流程

| 项目 | 内容 |
| --- | --- |
| 文档版本 | v0.1 |
| 适用范围 | MVP |
| 更新日期 | 2026-07-16 |

## 1. 文档目标

本文定义 MVP 的页面层级、导航关系、关键页面职责和核心用户流程，作为原型设计、前端路由和验收用例的共同依据。

## 2. 导航原则

- 以“当前宝宝”为全局上下文，首页、时间轴和成长数据均展示当前宝宝的数据。
- 高频任务最多在两次点击内开始：添加记录、切换宝宝、查看最近记录。
- 创建记录使用统一入口，根据类型进入不同表单。
- 管理类能力收纳在“我的”，避免干扰日常记录。
- 微信平台专属能力通过适配入口提供，不改变核心业务流程。

## 4. 底部导航

| 导航项 | 默认落点 | 核心任务 | 是否依赖当前宝宝 |
| --- | --- | --- | --- |
| 首页 | `/pages/home/index` | 查看概览、快速记录、切换宝宝 | 是 |
| 时间轴 | `/pages/timeline/index` | 浏览和筛选全部记录 | 是 |
| 成长数据 | `/pages/growth/index` | 查看身高、体重趋势 | 是 |
| 我的 | `/pages/profile/index` | 管理档案、成员、导出与设置 | 否 |

“添加记录”使用页面内突出按钮，不占用底部导航。详情页和表单页不显示底部导航。

## 5. 全局上下文与状态

### 5.1 当前宝宝

- 登录后若只关联一个宝宝，自动选中该宝宝。
- 若关联多个宝宝，默认恢复用户上次选择；无本地记录时选中最近加入的宝宝。
- 切换宝宝后刷新首页、时间轴和成长数据。
- 当前宝宝 ID 仅用于界面状态，服务端仍必须独立校验访问权限。
- 用户被移出当前家庭后，清除当前选择并跳转至可访问的宝宝或空状态。

### 5.2 无宝宝状态

当用户没有可访问的宝宝时，主界面展示：

- 创建宝宝档案按钮。
- 接受邀请的说明。
- 不展示空的时间轴和趋势图。

### 5.3 权限状态

| 操作 | 管理员 | 可编辑成员 | 只读成员 |
| --- | --- | --- | --- |
| 查看档案和记录 | 是 | 是 | 是 |
| 创建记录 | 是 | 是 | 否 |
| 编辑自己的记录 | 是 | 是 | 否 |
| 编辑他人的记录 | 是 | 否 | 否 |
| 删除自己的记录 | 是 | 是 | 否 |
| 删除他人的记录 | 是 | 否 | 否 |
| 编辑宝宝档案 | 是 | 否 | 否 |
| 邀请、授权、移除成员 | 是 | 否 | 否 |
| 导出宝宝数据 | 是 | 否 | 否 |

界面应根据权限隐藏或禁用操作，但服务端权限校验是最终保障。

## 6. 页面职责

### 6.1 登录与启动

页面目标：建立业务会话并确定用户的首个落点。

状态分支：

1. 未登录：展示微信登录与隐私协议确认。
2. 携带有效邀请：登录后进入邀请确认页。
3. 已登录且有关联宝宝：进入首页。
4. 已登录但无关联宝宝：进入创建宝宝引导。
5. 登录失败：展示重试，保留邀请参数。

### 6.2 首页

主要模块：

- 宝宝信息卡：头像、昵称、月龄和切换入口。
- 快捷记录：图文、测量、里程碑。
- 最近记录：展示最近 3 至 5 条，支持进入详情。
- 今日提醒：MVP 仅保留扩展位置；没有实际提醒时不展示虚假内容。

关键状态：加载中、无记录、加载失败、无编辑权限。

### 6.3 添加记录

统一入口先选择记录类型，也可由快捷入口直接带入类型。

| 类型 | 必填项 | 选填项 |
| --- | --- | --- |
| 图文 | 文字或至少一张照片、发生时间 | 位置描述（后续能力，MVP 不采集定位） |
| 测量 | 身高或体重至少一项、测量时间 | 备注、照片 |
| 里程碑 | 里程碑标题、发生时间 | 描述、照片、预设标签 |

保存时：

- 防止连续点击产生重复记录。
- 图片先完成上传或进入可恢复的上传状态，再提交业务记录。
- 成功后返回来源页面并刷新数据。
- 失败时保留表单内容，允许重试。

### 6.4 时间轴

- 默认按 `occurred_at` 倒序排列，相同时间按不可变记录 `id` 倒序，确保游标分页稳定。
- 支持“全部、图文、测量、里程碑”筛选。
- 使用游标分页，避免新增数据造成翻页重复或遗漏。
- 删除记录后从列表移除，并给出明确结果提示。
- MVP 不提供关键词搜索和按年月跳转。

### 6.5 记录详情

- 展示完整内容、照片、发生时间、创建者和记录类型。
- 根据权限展示编辑和删除操作。
- 删除必须二次确认。
- 图片使用临时授权地址，并支持预览。

### 6.6 成长数据

- 身高和体重使用分段切换展示。
- 默认显示最近 12 个月；数据不足时展示全部。
- 图表点与历史列表来自相同数据源。
- 点击图表点可查看数值和测量日期。
- 不展示诊断结论；异常输入仅作为数据校验提示。

### 6.7 我的

- 展示用户基础信息。
- 提供宝宝档案、家庭成员、数据导出、隐私政策和反馈入口。
- 多宝宝管理从宝宝档案列表进入。
- 管理入口按当前用户对所选宝宝的角色显示。

## 7. 核心用户流程

### 7.1 首次创建宝宝

```text
打开小程序
→ 同意隐私政策并微信登录
→ 系统发现没有关联宝宝
→ 填写宝宝必填信息
→ 创建宝宝和管理员成员关系
→ 自动选中新宝宝
→ 进入首页空状态
→ 引导添加第一条记录
```

异常处理：创建失败时保留表单；重复提交由客户端按钮锁和服务端幂等机制共同避免。

### 7.2 添加图文记录

```text
首页点击“图文”
→ 输入文字和/或选择照片
→ 确认发生时间
→ 上传照片
→ 保存记录
→ 返回首页
→ 最近记录展示新内容
```

用户取消且表单有内容时，提示是否放弃；MVP 不要求跨进程草稿恢复。

### 7.3 记录并查看成长数据

```text
点击“测量”
→ 输入身高和/或体重
→ 选择测量时间
→ 保存
→ 进入成长数据
→ 图表和历史列表同步出现数据点
```

### 7.4 邀请家庭成员

```text
管理员进入家庭成员
→ 选择目标角色
→ 生成一次性/限时邀请
→ 通过微信分享
→ 被邀请者打开邀请并登录
→ 查看宝宝昵称、邀请者和目标角色
→ 确认加入
→ 成为家庭成员
```

邀请失效、已使用、已是成员或管理员已撤销时，展示对应状态，不自动创建重复成员关系。

### 7.5 切换宝宝

```text
点击当前宝宝信息卡
→ 展示有权访问的宝宝列表
→ 选择宝宝
→ 更新全局当前宝宝
→ 返回原页面并加载对应数据
```

### 7.6 数据导出

```text
管理员进入数据导出
→ 选择导出范围
→ 确认隐私和下载说明
→ 创建异步导出任务
→ 查看处理中状态
→ 任务完成后获取限时下载地址
```

导出包格式和照片是否打包属于待确认事项。

## 8. 统一界面状态

每个数据页面至少考虑以下状态：

- 首次加载：骨架屏或明确加载指示。
- 空状态：说明原因并提供下一步操作。
- 局部刷新：保留已有数据，避免整页闪烁。
- 网络错误：提供重试，不清空用户输入。
- 无权限：说明权限已变化并刷新当前宝宝列表。
- 数据不存在：记录被删除后返回上级页面。
- 提交中：锁定提交按钮，避免重复请求。

## 9. 埋点建议

MVP 可先预留统一埋点接口，重点事件包括：

- 登录成功/失败。
- 宝宝档案创建成功。
- 三类记录创建成功/失败。
- 图片上传成功/失败。
- 家庭邀请生成、打开和接受。
- 宝宝切换。
- 数据导出创建和完成。

埋点不得包含宝宝姓名、记录正文、照片地址等敏感内容。

## 10. 待确认事项

- 首页是否在 MVP 展示“今日提醒”占位。
- 里程碑是否提供内置标签列表。
- 测量记录是否允许同时上传照片。
- 数据导出完成后的用户通知方式。
- 首次使用是否需要多页功能介绍，默认建议仅保留一步引导。
