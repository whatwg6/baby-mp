# 架构决策记录

ADR 用于记录难以从代码本身看出的重大工程决策。已经接受的决策在被新 ADR 替代前持续有效。

## 状态

- `Proposed`：提议中，不能作为稳定依赖。
- `Accepted`：已经接受，开发应遵循。
- `Superseded`：已由另一份 ADR 替代。
- `Deprecated`：不再推荐，但可能仍存在于旧实现中。

## 新增约定

- 文件名使用四位编号和英文短标题。
- 新 ADR 必须包含背景、决策、影响和替代方案。
- 改变既有决策时新增 ADR，并在双方文档中记录替代关系；不要直接抹去历史。
