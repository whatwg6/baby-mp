## 1. 测试目标

- 验证核心成长记录流程可用且数据一致。
- 重点验证宝宝数据不会在不同家庭、用户或宝宝之间越权泄漏。
- 验证弱网、重复提交、上传失败和会话失效不会造成数据丢失或重复。
- 验证微信小程序首发端的登录、拍照/选图、分享和真机表现。
- 为其他小程序和 H5 保留跨端回归基线。

## 2. 测试范围

### 2.1 MVP 范围内

- 登录、刷新和退出。
- 宝宝档案创建、查看、编辑和切换。
- 图文、测量、里程碑记录。
- 媒体上传、访问、排序和清理。
- 时间轴分页与筛选。
- 身高和体重趋势。
- 家庭邀请、角色修改和成员移除。
- 数据导出任务及限时下载。
- 权限、幂等、乐观并发、软删除和审计。
- 本地环境、微信开发者工具、微信真机和预发布环境。

### 2.2 MVP 范围外

- 喂养、睡眠、尿布和疫苗提醒。
- 视频、AI 内容和社区。
- 医疗判断及生长百分位。
- 支付功能。
- 正式支持支付宝、抖音和公开 H5 登录；只验证跨端架构未被明显破坏。

## 4. 测试环境

### 4.1 Local

- Taro H5 或微信开发者工具。
- NestJS 本地进程。
- Docker PostgreSQL。
- Docker MinIO。
- 模拟登录。
- 使用固定测试数据，不使用真实宝宝信息。

### 4.2 Test/CI

- 每次测试使用隔离数据库或独立 schema。
- MinIO 或兼容的临时对象存储。
- 固定系统时间能力，保证日期/月龄测试稳定。
- 禁止连接生产资源。

### 4.3 Staging

- 独立云数据库和私有对象存储。
- 真实微信 AppID、体验版和测试微信账号。
- 配置 HTTPS、合法域名和平台分享。
- 数据可定期清空，不与生产共用密钥。

### 4.4 Production smoke test

- 只使用专用测试宝宝和无敏感测试图片。
- 仅执行不会影响真实用户的最小冒烟流程。
- 禁止批量清理和破坏性测试。

## 5. 测试账号与数据

| 标识 | 角色/用途 |
| --- | --- |
| `parent-a` | 宝宝 A 的 admin、记录创建者 |
| `parent-b` | 宝宝 A 的 editor |
| `grandparent-a` | 宝宝 A 的 viewer |
| `outsider` | 不属于宝宝 A，用于越权测试 |
| `parent-multi` | 同时管理宝宝 B、C，用于切换测试 |

测试媒体至少包含：

- 合法 JPEG、PNG。
- 超大文件。
- MIME 与扩展名不一致文件。
- 损坏图片。
- 同名不同内容文件。

所有测试内容使用虚构姓名和合成图片。

## 6. 优先级与严重程度

### 6.1 用例优先级

- P0：隐私越权、数据损坏、核心链路阻塞。
- P1：MVP 主要功能、失败恢复、跨账号协作。
- P2：边界体验、非阻塞兼容性和优化。

### 6.2 缺陷严重程度

| 等级 | 定义 | 示例 |
| --- | --- | --- |
| S0 | 隐私/安全事故或不可恢复数据破坏 | outsider 可读取宝宝照片 |
| S1 | 核心流程不可用或大范围错误 | 无法创建记录、记录串宝宝 |
| S2 | 部分功能失败且有替代路径 | 某筛选失效、导出需重试 |
| S3 | 轻微视觉或文案问题 | 间距不一致、非关键文案错误 |

## 8. 客户端交互用例

### 8.1 表单

- 提交失败后保留字段、照片列表和发生时间。
- 有未保存修改时返回，显示放弃确认。
- 提交中按钮锁定。
- 首个错误字段可见并获得焦点或滚动定位。
- viewer 看不到新建入口；权限改变后提交得到明确提示。

### 8.2 页面状态

- 首次加载、空状态、局部刷新、分页失败和整页失败分别验证。
- 无权限状态不短暂闪现缓存数据。
- 记录在其他设备删除后，详情显示安全状态并允许返回。
- 多宝宝切换清理列表、图表和详情缓存键。

### 8.3 图片

- 拍照和相册选择均可用。
- 预览、删除、排序和单张重试可用。
- 上传中离开页面有明确提示。
- 临时签名 URL 失效后可重新获取，不把失效 URL 永久缓存。

## 9. 端到端关键链路

### E2E-01 首次使用

```text
模拟/微信登录
→ 创建宝宝
→ 首页空状态
→ 创建图文记录
→ 首页出现记录
→ 详情内容一致
```

### E2E-02 成长数据

```text
创建含身高体重的测量
→ 成长页查看两个指标
→ 编辑数值
→ 图表和列表更新
→ 删除记录
→ 数据点消失
```

### E2E-03 家庭协作

```text
admin 创建 editor 邀请
→ 第二账号接受
→ editor 创建记录
→ admin 可编辑该记录
→ editor 不可编辑 admin 记录
→ admin 将其改为 viewer
→ 该用户不能再创建记录
```

### E2E-04 多宝宝隔离

```text
同一用户关联宝宝 A、B
→ 在 A 创建记录和图片
→ 切换 B
→ 首页、时间轴、成长页均无 A 数据
→ 尝试用 A mediaId 创建 B 记录被拒绝
```

### E2E-05 数据导出

```text
admin 创建包含照片的导出
→ 等待任务完成
→ 获取限时下载地址
→ 核对档案、记录和图片归属
→ 地址过期后访问失败
```

### E2E-06 数据权利与立即失权

```text
登录后提交宝宝范围数据访问申请
→ 页面显示待人工核验
→ 取消后显示已取消
→ 第二个宝宝范围申请保持 pending
→ 用户退出家庭
→ 页面和 API 不再显示该宝宝数据
→ 重放旧范围申请被拒绝
```

### E2E-07 弱网与恢复

```text
宝宝列表第一次返回 503
→ 客户端有界重试
→ 页面恢复且不丢宝宝上下文

快速连续点击两次保存
→ 使用同一幂等键
→ 数据库只产生一条记录

受保护请求与 refresh 同时返回 401
→ 清空会话和当前宝宝缓存
→ 返回登录页且不残留宝宝数据

图片预签名 PUT 连接被重置
→ 文字和图片草稿保留
→ 显示失败和重试入口
```

上述四条已纳入 H5 Playwright；微信拍照/相册授权、前后台切换和系统回收仍按真机清单验收。

## 10. 微信真机检查

- 微信登录 code 交换成功，session_key 不出现在客户端或日志。
- 拍照、相册授权拒绝与重新授权行为清楚。
- 上传合法域名、下载合法域名配置正确。
- 分享路径携带邀请 token，冷启动和已登录两种场景均可恢复。
- 胶囊按钮、安全区域和底部导航无遮挡。
- iOS 和 Android 至少各一台设备验证。
- 前后台切换、网络断开恢复和小程序被系统回收后的行为可接受。
- 包体大小、首屏时间和图表渲染符合微信限制。

## 11. 非功能检查

### 11.1 性能基线

具体阈值在实现后根据预发布测量确定，初始目标：

- 普通 API 在测试环境的 P95 响应时间低于 500 ms（不含上传和导出）。
- 时间轴首屏最多请求 20 条记录。
- 图片列表使用缩略图和懒加载。
- 1000 条记录的宝宝仍可稳定分页。
- 趋势图 500 个点时界面仍可交互；必要时按时间范围或抽样展示。

### 11.2 安全基线

- 所有宝宝域接口均有资源级授权测试。
- 对象存储桶不允许匿名列举和读取。
- token、平台密钥、原始邀请 token、宝宝正文和签名 URL 不进入日志。
- 依赖扫描没有未接受的高危漏洞。
- 错误响应不返回堆栈、数据库 SQL 或云配置。

### 11.3 备份恢复

- 预发布至少完成一次 PostgreSQL 备份和恢复。
- 验证数据库恢复后媒体引用仍可解析。
- 验证迁移失败时的恢复步骤。

## 12. 发布准入条件

- P0 用例全部自动化或有明确的发布前人工记录，并全部通过。
- P1 核心用例通过率 100%。
- 无未关闭 S0/S1 缺陷。
- 权限矩阵和多宝宝隔离测试通过。
- 微信真机核心链路通过。
- 数据库备份恢复演练通过。
- 隐私政策和微信隐私接口声明与实际收集行为一致。
- 监控、告警和回滚步骤可执行。
