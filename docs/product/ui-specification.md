## 1. 使用说明

本文规定页面结构、用户操作、表单校验、权限差异和异常状态。开发 agent 可以基于本文实现低保真界面，但不应把示意布局视为最终品牌视觉。

若本文与其他文档冲突，按以下优先级处理：

1. 产品范围与业务规则：`docs/product/product-requirements.md`
2. 页面结构与流程：`docs/product/information-architecture.md`
3. 页面具体行为：本文
4. API 字段：`docs/architecture/api-specification.md`
5. 数据库实现：`docs/architecture/data-model.md`

## 2. 体验原则

- 温和、克制、私密，不使用公开社区式的关注、热度或排名表达。
- 高频记录路径短，默认值合理，失败后不丢失输入。
- 宝宝照片是视觉重点，但任何图片都不能使用永久公开地址。
- 不将成长数据包装为医疗诊断，不用红色制造健康焦虑。
- 管理和隐私操作清楚说明后果，危险操作二次确认。
- 老年家庭成员也应能完成查看和简单记录，点击区域和字号不能过小。

## 3. 全局布局与导航

### 3.1 底部导航

| 标签 | 路由 | 图标语义 |
| --- | --- | --- |
| 首页 | `/pages/home/index` | 房屋/家庭 |
| 时间轴 | `/pages/timeline/index` | 时间/相册 |
| 成长 | `/pages/growth/index` | 趋势图 |
| 我的 | `/pages/profile/index` | 用户/设置 |

- 首页为默认选中项。
- 详情、编辑、邀请确认和导出详情页隐藏底部导航。
- 未登录时不能进入主框架。
- 无宝宝时允许进入“我的”，其余三个入口显示统一引导状态。

### 3.2 顶部安全区域

- 使用 Taro 提供的系统信息适配状态栏和胶囊按钮。
- 不在页面中写死微信状态栏高度。
- H5 使用普通导航栏；标题和返回行为与小程序一致。

### 3.3 全局反馈

| 场景 | 反馈方式 |
| --- | --- |
| 短操作成功 | Toast，约 1.5 秒 |
| 保存、上传等进行中 | 按钮 loading，必要时显示进度 |
| 普通错误 | 页面内错误或 Toast，提供重试 |
| 删除、退出家庭 | 二次确认弹窗 |
| 无权限 | 明确说明权限变化并返回安全页面 |
| 会话失效 | 尝试一次重新登录，失败后进入登录页 |

## 4. 通用组件

### 4.1 宝宝信息卡 `BabyHeaderCard`

内容：头像、昵称、月龄、当前成员角色、多宝宝切换箭头。

- 点击整张卡打开宝宝切换器。
- 只有一个宝宝时仍可点击，切换器中提供“管理宝宝档案”。
- 月龄按当前日期动态计算；满 24 个月后可显示“2 岁 3 个月”。

### 4.2 宝宝切换器 `BabySwitcher`

使用底部抽屉：

- 列出所有可访问且未删除的宝宝。
- 当前宝宝显示选中标记。
- 每项展示头像、昵称和用户角色。
- 底部提供“创建宝宝”和“管理宝宝档案”。
- 切换时关闭抽屉并刷新当前页面。

### 4.3 记录卡 `RecordCard`

共有内容：记录类型、发生日期、创建者、文字摘要和媒体缩略图。

- 图文记录优先展示首张图。
- 测量记录突出显示身高和/或体重。
- 里程碑记录突出显示标题。
- 卡片整体可点击进入详情。
- 列表卡片不展示编辑和删除按钮，避免误触。

### 4.4 页面状态 `PageState`

统一支持：`loading`、`empty`、`error`、`forbidden`。

- `empty` 必须说明原因，并在用户有权限时提供主要操作。
- `error` 保留已加载内容；首次加载失败时才使用整页错误。
- `forbidden` 不显示资源的任何残留内容。

### 4.5 确认弹窗 `ConfirmDialog`

- 明确操作对象和结果。
- 删除等危险操作的主要按钮使用危险色。
- 不使用含糊的“确定/取消”作为全部文案，例如使用“删除记录/暂不删除”。

## 5. 页面规格

### 5.1 启动与登录页

路由：`/pages/auth/index`

页面目标：完成隐私确认和平台登录，保持邀请上下文。

内容：

- 产品名称和简洁说明。
- “微信登录”主按钮；本地环境显示“以测试用户登录”。
- 隐私政策和用户协议确认项。
- 邀请进入时展示邀请者和宝宝的安全摘要，不展示成长内容。

交互规则：

- 未勾选协议时点击登录，定位并提示协议确认，不直接登录。
- 登录按钮提交中不可重复点击。
- 登录失败展示可重试错误，保留邀请 token。
- 登录成功后按“邀请确认 → 已有宝宝首页 → 创建宝宝引导”顺序决定落点。
- `production` 不得展示模拟登录入口。

### 5.2 创建宝宝引导

路由：`/pages/babies/create`

低保真结构：

```text
[返回] 创建宝宝档案

       [头像]
宝宝昵称  [____________]
性别      [男] [女] [暂不设置]
出生日期  [选择日期 >]
出生时间  [选填 >]
出生身高  [选填____ cm]
出生体重  [选填____ kg]

[创建宝宝]
```

校验：

- 昵称：去除首尾空格后 1–40 个字符。
- 出生日期：必填，不晚于设备当前日期。
- 出生时间：选填，仅保存时分。
- 出生身高：选填，20–250 cm，最多两位小数。
- 出生体重：选填，0.2–300 kg，最多三位小数。
- 头像：选填；未上传时使用本地默认头像。

保存行为：

- 成功后选择新宝宝并进入首页。
- 失败时保留所有字段。
- 使用幂等键避免重复创建。
- 创建页选择的头像先作为本地预览；宝宝创建成功后，再以新 `babyId` 上传并更新头像。头像上传失败不回滚宝宝档案，首页使用默认头像并提供重试。

### 5.3 首页

路由：`/pages/home/index`

低保真结构：

```text
[宝宝信息卡                         >]

快速记录
[图文]        [测量]        [里程碑]

最近记录                         [查看全部]
[记录卡]
[记录卡]
[记录卡]
```

规则：

- 最近记录默认最多展示 5 条。
- “查看全部”进入时间轴并保持“全部”筛选。
- 快捷按钮直接进入对应记录表单，不再显示类型选择。
- viewer 仍可查看首页，但隐藏快速记录区域。
- 无记录时展示“记录宝宝的第一个成长瞬间”；editor/admin 可点击创建，viewer 仅显示说明。
- MVP 没有真实提醒能力时不展示“今日提醒”模块。

### 5.4 记录类型选择器

入口：首页或时间轴的通用“添加”按钮。

使用底部抽屉展示：

- 图文记录：照片、文字和日常瞬间。
- 身高体重：一次测量可填写一项或两项。
- 成长里程碑：第一次、纪念日等重要事件。

viewer 不显示添加入口。角色变化导致打开后失去权限时，提交由服务端拒绝并返回安全页面。

### 5.5 新建/编辑图文记录

路由：

- 新建：`/pages/records/edit?type=note&babyId=:babyId`
- 编辑：`/pages/records/edit?id=:recordId`

字段：

- 正文：最多 2000 个字符。
- 照片：0–9 张，可预览、删除和排序。
- 发生时间：默认当前时间，不早于宝宝出生时间，不晚于当前时间 5 分钟以上。

规则：

- 正文和照片至少有一项。
- 选择照片后立即进入待上传列表；保存时确保每张照片已变为 `ready`。
- 上传失败的照片可单独重试或删除。
- 退出且有修改时提示是否放弃。
- 保存成功返回来源页；来源不可判断时返回记录详情。

### 5.6 新建/编辑测量记录

字段：

- 身高：20–250 cm，选填。
- 体重：0.2–300 kg，选填。
- 备注：最多 500 个字符，选填。
- 照片：0–9 张，选填。
- 测量时间：默认当前时间。

规则：

- 身高和体重至少填写一项。
- 输入使用十进制数字键盘，不允许科学计数法。
- 不自动换算斤；MVP 只使用千克。
- 页面显示“成长数据仅用于家庭记录，不构成医疗建议”。
- 照片沿用统一上传、预览、删除、排序和失败重试能力。

### 5.7 新建/编辑里程碑

字段：

- 标题：必填，1–60 个字符。
- 描述：选填，最多 2000 个字符。
- 照片：0–9 张。
- 发生时间：默认当前时间。

MVP 不提供固定里程碑标签库，避免标签和年龄阶段规则阻塞首版。后续可在标题输入上方增加推荐标签。

### 5.8 时间轴

路由：`/pages/timeline/index`

结构：

- 顶部当前宝宝简化信息与切换入口。
- 横向筛选：全部、图文、测量、里程碑。
- 记录列表。
- editor/admin 的悬浮添加按钮。

规则：

- 筛选变化后回到列表顶部并重新加载。
- 首次加载使用骨架屏，分页加载使用底部 loading。
- 下拉刷新保留当前筛选。
- 分页失败保留已有列表并显示“重新加载”。
- 页面返回时，如果详情发生编辑或删除，刷新受影响项。
- 列表封面启用平台图片懒加载，避免首屏同时请求全部签名图片。
- 当前媒体契约只提供原图短时授权地址，不提供服务端缩略图。客户端不得通过改写对象路径或签名参数伪造缩略图地址；后续需先增加独立缩略图对象、授权和契约字段，才能让时间轴请求真实缩略图。

### 5.9 记录详情

路由：`/pages/records/detail?id=:recordId`

展示：

- 类型、标题/正文、完整图片、发生时间、创建者。
- 测量记录展示身高和/或体重。
- 创建时间不作为主要信息，仅在需要时显示。

操作：

- admin 可编辑或删除任何记录。
- editor 只可编辑或删除自己的记录。
- viewer 无操作菜单。
- 删除确认文案包含“删除后将从时间轴和成长曲线中移除”。
- 资源已删除或无权访问时清空页面并返回安全状态。

### 5.10 成长数据

路由：`/pages/growth/index`

结构：

```text
[当前宝宝                         >]
[身高] [体重]

[趋势图区域]

历史记录                         [+ 新增]
2026-07-16     68.2 cm
2026-06-10     65.0 cm
```

规则：

- 默认指标为最近查看的指标；首次进入默认身高。
- 默认范围为最近 12 个月，数据不足时显示全部。
- 单点数据仍显示点和数值，不绘制误导性趋势。
- 点击图表点显示日期和精确值。
- editor/admin 显示新增入口，viewer 隐藏。
- 历史项点击进入记录详情。
- 图表不展示百分位、正常/异常或预测结论。

### 5.11 我的

路由：`/pages/profile/index`

分组：

1. 用户信息。
2. 宝宝档案。
3. 家庭成员。
4. 数据导出。
5. 隐私政策、用户协议和反馈。
6. 退出登录。

权限：

- 用户可编辑自己的显示名；trim 后为 1–80 个 Unicode 字符，失败时保留输入并允许重试。
- 所有人可查看宝宝档案摘要。
- 只有 admin 可编辑档案、管理成员和导出。
- 无宝宝时仅展示创建宝宝、协议、反馈和退出登录。
- 内部测试期的反馈入口引导用户回到测试邀请消息或测试群联系版本发放者，并附设备、时间、复现步骤和截图；在运营主体和正式联系方式未确认前不得虚构客服电话、邮箱或处理承诺。

### 5.12 宝宝档案列表与编辑

路由：

- 列表：`/pages/babies/index`
- 编辑：`/pages/babies/edit?id=:babyId`

列表显示用户可访问的全部宝宝和角色。只有 admin 在对应项上看到“编辑”。

编辑页沿用创建页校验。删除宝宝属于高风险能力：

- MVP 可先显示“申请删除”入口，不要求立即物理删除。
- 必须说明家庭成员将无法访问、导出期限和数据清理规则。
- 最终交互需在隐私政策完成后确认。

### 5.13 家庭成员列表

路由：`/pages/family/members?babyId=:babyId`

内容：

- 当前宝宝摘要。
- 成员头像、显示名、角色和加入时间。
- admin 的“邀请成员”按钮。
- admin 可修改其他成员角色或移除成员。

规则：

- MVP 邀请角色只允许 editor 或 viewer。
- 最后一个 admin 不能降级、退出或被移除。
- 用户操作自己的 admin 角色时说明先指定另一管理员；MVP 暂不提供邀请 admin，但可将现有成员提升为 admin。
- 移除成员必须二次确认，并说明其将立即失去访问权限。

### 5.14 创建和接受邀请

路由：

- 创建：`/pages/family/invite-create?babyId=:babyId`
- 接受：`/pages/family/invite?token=:token`

创建邀请：

- 选择“可编辑”或“只读”。
- 默认有效期 24 小时。
- 生成后通过平台分享适配器发送。
- 可在成员页查看并撤销尚未接受的邀请。

接受邀请页展示：

- 宝宝昵称和头像。
- 邀请者显示名。
- 加入后的权限说明。
- “加入家庭”和“暂不加入”。

状态：有效、已接受、已撤销、已过期、已经是成员。无效状态不展示宝宝成长数据。

页面路由中的邀请 token 只用于客户端恢复邀请上下文；调用 API 时放入 JSON 请求体。客户端、服务端和代理日志必须对该参数脱敏。

### 5.15 数据导出

路由：

- 列表：`/pages/exports/index?babyId=:babyId`
- 详情：`/pages/exports/detail?id=:exportId`

MVP 默认选项：

- 档案、文字、里程碑和测量数据始终包含。
- 文件固定为 ZIP，内含规范 JSON 与 UTF-8 CSV。
- 是否包含照片由用户勾选，默认不勾选。

状态显示：等待处理、处理中、已完成、失败、已过期。

- 创建任务前说明可能耗时和下载链接有效期。
- 完成后显示“获取下载链接”。
- 失败显示错误摘要和重新创建入口。
- 过期后不复用旧链接，允许重新导出。

## 6. 表单与数据规则汇总

| 字段 | 规则 |
| --- | --- |
| 用户显示名 | trim 后 1–80 个 Unicode 字符 |
| 宝宝昵称 | 必填，trim 后 1–40 字符 |
| 出生日期 | 必填，不晚于今天 |
| 出生身高 | 20–250 cm，最多 2 位小数 |
| 出生体重 | 0.2–300 kg，最多 3 位小数 |
| 图文正文 | 0–2000 字符，与照片至少有一项 |
| 里程碑标题 | 1–60 字符 |
| 里程碑描述 | 0–2000 字符 |
| 测量备注 | 0–500 字符 |
| 照片 | 0–9 张；单张实际上传内容不超过 20 MiB |
| 发生时间 | 不早于宝宝出生日期与可选精确出生时间，不明显晚于当前时间 |

客户端和服务端必须使用相同规则；服务端是最终校验方。H5 保留浏览器选中的原始 JPEG/PNG，
小程序可在上传前使用平台压缩；两端都以实际上传字节执行 20 MiB 硬上限。

## 7. 弱网与草稿策略

- MVP 不实现跨设备草稿同步。
- 页面进程仍在时，上传失败或保存失败不得清空输入。
- H5 使用可报告真实字节进度的上传实现；微信小程序的签名原始 `PUT` 不提供上传字节进度，界面只显示“上传中”和真实完成状态，不伪造百分比。
- 微信小程序上传前压缩并以 20 MB 契约上限约束单次内存读取；上传任务支持取消，页面隐藏、卸载或删除待上传项时终止进行中的网络请求。
- 用户主动退出有修改的表单时必须确认。
- 保存按钮第一次点击后立即锁定，写请求携带幂等键。
- 应用重新启动后的草稿恢复为 P2，不属于 MVP 验收项。

## 8. 可访问性与适老性

- 正文默认字号不低于 16 CSS px 对应视觉尺寸。
- 主要点击区域建议不小于 44×44 CSS px。
- 颜色不能成为区分状态的唯一方式。
- 图表数据同时提供文字列表。
- 图片提供简短类型描述，不要求用户为每张家庭照片手写替代文本。
- 表单错误显示在对应字段附近，并在提交后聚焦首个错误。

## 9. 视觉待定项

以下内容由后续视觉设计确定，不阻塞低保真实现：

- 正式品牌名、Logo 和启动图。
- 主色、辅助色和插画风格。
- 字体层级和精确间距 token。
- 默认宝宝头像和空状态插画。
- 图标库及平台差异。

开发时应使用语义 token，例如 `color-primary`、`color-danger`、`space-md`，不要把临时颜色散落在业务组件中。
