## 1. 产品概述

### 1.1 产品定位

“宝宝成长记”是一款面向家庭的宝宝成长记录工具，帮助父母通过文字、照片、成长数据和里程碑长期记录宝宝的成长过程，并邀请家庭成员共同参与。

产品以微信小程序为首发平台，同时在技术架构上保留编译到支付宝小程序、抖音小程序和 H5 的能力。

### 1.2 产品目标

- 降低日常成长记录的操作成本，让用户可以快速完成一次记录。
- 将照片、文字、成长数据和重要时刻集中到统一时间轴。
- 支持家庭成员共同记录，同时保护宝宝及家庭的隐私。
- 帮助父母直观了解身高、体重等数据的变化趋势。
- 确保用户可以备份和导出自己的数据。

### 1.3 暂不纳入 MVP 的目标

- 社区、公开动态或陌生人社交。
- 在线问诊、疾病诊断或医疗建议。
- 复杂的育儿电商和广告系统。
- 完整的喂养、睡眠和尿布量化管理。
- AI 自动生成周报、月报和纪念册。

## 2. 目标用户与角色

### 2.1 目标用户

- 爸爸、妈妈：宝宝成长内容的主要记录者。
- 爷爷奶奶、外公外婆等家庭成员：查看宝宝动态，或在获得权限后共同记录。
- 多孩家庭：在同一账号下分别管理多个宝宝档案。

### 2.2 家庭角色

| 角色 | 默认权限 |
| --- | --- |
| 创建者/管理员 | 管理宝宝档案、邀请和移除成员、创建记录、管理全部记录、导出数据 |
| 可编辑成员 | 查看全部家庭记录、创建记录、修改或删除自己创建的记录 |
| 只读成员 | 查看宝宝档案和成长记录，不可创建或修改内容 |

## 3. 核心使用场景

1. 妈妈拍摄宝宝第一次走路的照片，并添加文字和“第一次走路”里程碑。
2. 爸爸记录宝宝本次体检的身高和体重，并查看历史变化趋势。
3. 父母邀请祖辈加入家庭，让他们查看宝宝最近的成长动态。
4. 多孩家庭在首页切换宝宝，分别查看和记录每个宝宝的数据。
5. 家庭管理员导出宝宝的成长记录，进行长期备份。

## 4. MVP 功能范围

### 4.1 微信登录与首次使用

- 用户通过微信完成小程序登录。
- 首次登录后引导用户创建宝宝档案。
- 已受邀用户可以接受家庭邀请并进入对应宝宝空间。
- 为其他平台预留平台账号与统一账号绑定能力。

### 4.2 宝宝档案

- 创建、查看和编辑宝宝档案。
- 支持字段：
  - 姓名或昵称
  - 头像
  - 性别
  - 出生日期
  - 出生时间（选填）
  - 出生身高（选填）
  - 出生体重（选填）
- 一个账号可以创建或加入多个宝宝空间。
- 支持在多个宝宝之间切换。

### 4.3 成长记录

MVP 支持三类成长记录：

1. 普通图文记录
2. 身高、体重测量记录
3. 成长里程碑记录

通用能力包括：

- 输入文字内容。
- 从相册选择或拍摄照片。
- 自动填写当前时间，也允许修改记录时间。
- 显示记录创建者。
- 查看、编辑和删除记录。
- 上传前压缩图片，并保留后续支持原图存储策略的能力。

### 4.4 成长时间轴

- 按记录时间倒序展示宝宝的成长记录。
- 展示照片缩略图、文字摘要、记录类型、日期和创建者。
- 支持进入记录详情页。
- 支持按记录类型筛选。
- 切换宝宝后，仅展示当前宝宝的数据。

### 4.5 成长数据

- 展示身高趋势图。
- 展示体重趋势图。
- 展示历史测量记录列表。
- 支持新增、修改和删除测量数据。
- 明确标注单位：身高使用厘米，体重使用千克。
- MVP 仅展示用户记录的数据变化，不提供医疗诊断结论。

### 4.6 家庭协作

- 管理员通过微信分享或邀请入口邀请家庭成员。
- 被邀请者确认后加入对应宝宝空间。
- 管理员可以设置成员为“可编辑”或“只读”。
- 管理员可以移除家庭成员。
- 每条记录显示创建者，便于家庭协作追溯。

### 4.7 数据导出与隐私

- 用户可以发起宝宝成长数据导出。
- 导出内容至少包含宝宝档案、文字记录、里程碑和测量数据。
- 照片导出方式根据存储成本和生成文件大小在技术设计阶段确定。
- 宝宝数据默认仅对已加入的家庭成员可见。
- 删除宝宝档案、成员或大量数据时必须二次确认。
- 用户退出家庭后，不再具有该宝宝数据的访问权限。

## 5. 页面与导航

MVP 建议采用四个底部导航入口：

| 页面 | 主要内容 |
| --- | --- |
| 首页 | 宝宝信息、最近记录、快捷记录入口、提醒占位 |
| 时间轴 | 全部成长记录、类型筛选、记录详情 |
| 成长数据 | 身高和体重趋势、历史测量数据 |
| 我的 | 宝宝档案、家庭成员、数据导出、隐私设置、意见反馈 |

“添加记录”作为首页和时间轴中的突出操作入口，不单独占用底部导航位置。

### 5.1 首页

- 当前宝宝的头像、姓名和月龄。
- 多宝宝切换入口。
- 添加图文、测量和里程碑的快捷入口。
- 最近成长记录摘要。
- 今日提醒区域；MVP 可先作为后续功能入口或空状态展示。

### 5.2 添加记录页

- 选择记录类型。
- 填写对应记录内容。
- 添加照片。
- 选择记录时间。
- 保存、取消和异常重试。

### 5.3 时间轴与详情页

- 时间轴列表及筛选器。
- 记录详情。
- 根据权限显示编辑和删除操作。

### 5.4 成长数据页

- 身高与体重指标切换。
- 趋势图。
- 历史记录列表。
- 新增测量数据入口。

### 5.5 我的页面

- 宝宝档案管理。
- 家庭成员与权限管理。
- 数据导出。
- 隐私说明。
- 意见反馈。

## 6. 核心业务规则

- 一个用户可以关联多个宝宝，一个宝宝可以关联多个用户。
- 宝宝档案创建者默认成为管理员。
- 管理员可以管理全部记录；普通可编辑成员只能修改或删除自己创建的记录。
- 只读成员不能新增、编辑或删除任何宝宝数据。
- 所有列表和统计均以记录发生时间为准，而不是数据创建时间。
- 身高和体重数据必须为正数，并设置合理的输入范围和异常提示。
- 删除记录采用软删除或保留审计信息，具体方式在数据设计阶段确定。
- 家庭邀请应包含有效期，并避免邀请链接被无限传播使用。
- 产品不向未加入家庭的用户公开宝宝数据。

## 7. 非功能需求

### 7.1 隐私与安全

- 所有接口必须校验用户对宝宝空间的访问权限。
- 照片不得使用可永久公开访问的固定地址。
- 敏感操作需要二次确认，并保留必要的操作记录。
- 遵循微信平台规则及适用的个人信息保护要求。
- 隐私政策需要说明收集的数据类型、用途、存储方式和删除方式。

### 7.2 性能与体验

- 常用页面应在正常移动网络下快速展示可交互内容。
- 时间轴采用分页加载和图片懒加载。
- 图片上传应显示进度，并支持失败重试。
- 弱网情况下避免重复创建记录。
- 空状态、加载状态和错误状态均需提供明确反馈。

### 7.3 跨端兼容

- 业务组件优先使用 Taro 跨端 API。
- 微信登录、分享等平台能力通过适配层封装。
- 避免业务代码直接依赖微信专有 API。
- 各平台构建产物需要独立完成核心流程验证。

## 8. 技术方案建议

### 8.1 客户端

- 跨端框架：Taro
- UI 框架：React
- 开发语言：TypeScript
- 状态管理：Zustand
- 图表：ECharts 或兼容 Taro 的图表封装
- UI 组件：NutUI Taro 或自建组件库

### 8.2 服务端

- 服务框架：Node.js + NestJS
- 数据库：PostgreSQL
- 文件存储：兼容 S3 协议的对象存储
- 登录方式：微信小程序登录，服务端签发业务会话
- 部署方式：Docker + 云服务器或云托管

采用独立后端可以降低对微信云开发的绑定，更适合后续扩展到其他小程序平台和 H5。

## 9. 后续版本候选功能

- 喂奶、辅食、换尿布和睡眠记录。
- 疫苗、体检和纪念日提醒。
- 视频记录。
- WHO 或中国儿童生长标准曲线。
- AI 成长周报、月报和年度总结。
- 自动生成成长相册和纪念册。
- 更完整的数据备份、恢复和迁移。
- 家庭互动、点赞和评论。

## 10. MVP 验收标准

MVP 达到以下条件后可进入首轮试用：

- 新用户可以完成微信登录并创建宝宝档案。
- 用户可以创建图文、测量和里程碑记录。
- 时间轴能够正确展示、筛选和查看记录。
- 身高、体重趋势图与历史数据一致。
- 管理员可以邀请、授权和移除家庭成员。
- 不同权限的成员只能执行被授权的操作。
- 用户可以切换多个宝宝，且数据不会串联。
- 图片上传失败时可以重试，不会产生重复记录。
- 用户可以发起数据导出。
- 核心流程在微信小程序端通过测试。

## 11. 待确认事项

- 产品正式名称和视觉风格。
- 首批用户主要覆盖的宝宝年龄段。
- 是否在 MVP 中加入疫苗或体检提醒。
- 照片是否需要保存原图，以及相应存储成本策略。
- 数据导出的文件格式和照片打包方式。
- 首发部署区域、云服务商和合规要求；该选择不阻塞本地开发，最晚在预发布环境建设前确定。
